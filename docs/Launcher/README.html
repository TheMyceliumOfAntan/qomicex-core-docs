启动器模块 (Launcher)

负责 Minecraft 游戏启动管理，包括参数准备、进程启动、日志记录和崩溃检测。

概述

Launcher 类是 Qomicex.Core 的核心组件，负责完整的游戏启动流程。它处理从准备启动参数到管理游戏进程的所有任务，并提供实时日志输出和崩溃检测功能。

命名空间

using Qomicex.Core.Modules.Launcher;

Launcher 类

事件

事件名 类型 说明
OnOutput Action<string>? 游戏标准输出日志事件
OnError Action<string>? 游戏错误输出事件
OnExited Action<int>? 进程退出事件（参数为退出代码）
OnCrashDetected Action<string>? 崩溃检测事件

方法

Launch

启动游戏进程，包含完整的启动前检查和准备。

public bool Launch(LauncherParam param)

参数： | 参数名 | 类型 | 说明 | |--------|------|------| | param | LauncherParam | 启动参数 |

返回值： bool - 启动是否成功

启动流程：

  1. 准备日志文件
  2. 检查 Java 版本是否符合 Minecraft 版本要求
  3. 检查内存分配是否合理
  4. 解压 Natives 文件
  5. 生成完整启动参数
  6. 创建并启动游戏进程
  7. 设置进程输出监听和错误监听
  8. 启用进程退出和崩溃检测

示例：

var launcher = new Launcher();

// 订阅事件
launcher.OnOutput += (log) => Console.WriteLine($"[输出] {log}");
launcher.OnError += (err) => Console.WriteLine($"[错误] {err}");
launcher.OnExited += (code) => Console.WriteLine($"进程退出，代码: {code}");
launcher.OnCrashDetected += (report) => Console.WriteLine($"检测到崩溃: {report}");

// 准备启动参数
var param = new LauncherParam
{
    GameDir = @"C:\Minecraft\.minecraft",
    LauncherName = "MyLauncher",
    Version = "1.20.1",
    MaxMemory = "4096",
    FullScreen = false,
    Width = "1920",
    Height = "1080",
    Java = new DataModules.DataDetails.Java
    {
        Path = @"C:\Program Files\Java\jdk-17\bin\java.exe",
        Version = "17.0.2",
        VersionID = 17
    },
    Account = new DataModules.DataDetails.Account
    {
        Name = "Steve",
        Uuid = "...",
        LoginMethod = "Offline"
    }
};

// 启动游戏
bool success = launcher.Launch(param);
if (success)
{
    Console.WriteLine("游戏启动成功！");
}
else
{
    Console.WriteLine("游戏启动失败！");
}

SelectParam

生成完整的 Java 启动参数字符串。

public string SelectParam(LauncherParam param)

参数： | 参数名 | 类型 | 说明 | |--------|------|------| | param | LauncherParam | 启动参数 |

返回值： string - 完整的 Java 启动参数字符串

参数生成流程：

  1. 读取并解析版本 JSON 文件
  2. 处理继承版本的参数
  3. 合并 JVM 参数和游戏参数
  4. 处理 Java 版本兼容性问题
  5. 处理 OptiFine 与 Forge 兼容性
  6. 替换变量占位符

UnzipNatives

解压并提取版本所需的 Natives 文件。

public bool UnzipNatives(string jsonPath, string minecraftPath, string versionPath)

参数： | 参数名 | 类型 | 说明 | |--------|------|------| | jsonPath | string | 版本 JSON 文件路径 | | minecraftPath | string | .minecraft 目录路径 | | versionPath | string | 版本文件夹路径 |

返回值： bool - 解压是否成功

示例：

var launcher = new Launcher();
bool success = launcher.UnzipNatives(
    @"C:\Minecraft\.minecraft\versions\1.20.1\1.20.1.json",
    @"C:\Minecraft\.minecraft",
    @"C:\Minecraft\.minecraft\versions\1.20.1"
);

LauncherParam 类

启动参数类，继承自 DataDetails.Launcher。

public class LauncherParam : DataDetails.Launcher
{
    public string GameDir = string.Empty;
    public string LauncherName = string.Empty;
    public new DataDetails.Java Java = new DataDetails.Java();
    public bool FullScreen = false;
    public string Width = "854";
    public string Height = "480";
}

属性说明： | 属性名 | 类型 | 默认值 | 说明 | |--------|------|--------|------| | GameDir | string | string.Empty | 游戏目录路径 | | LauncherName | string | string.Empty | 启动器名称（用于标识） | | Java | DataDetails.Java | new DataDetails.Java() | Java 运行时信息 | | FullScreen | bool | false | 是否全屏启动 | | Width | string | "854" | 游戏窗口宽度 | | Height | string | "480" | 游戏窗口高度 |

完整使用示例

using System;
using System.Threading;
using Qomicex.Core.Modules.Launcher;
using Qomicex.Core.DataModules;

class Program
{
    static void Main(string[] args)
    {
        // 创建启动器实例
        var launcher = new Launcher();

        // 订阅事件以接收游戏输出和状态
        launcher.OnOutput += (log) =>
        {
            Console.ForegroundColor = ConsoleColor.Gray;
            Console.WriteLine($"[游戏输出] {log}");
            Console.ResetColor();
        };

        launcher.OnError += (err) =>
        {
            Console.ForegroundColor = ConsoleColor.Red;
            Console.WriteLine($"[错误] {err}");
            Console.ResetColor();
        };

        launcher.OnExited += (code) =>
        {
            Console.ForegroundColor = ConsoleColor.Yellow;
            Console.WriteLine($"\n游戏进程已退出，退出代码: {code}");
            Console.ResetColor();
        };

        launcher.OnCrashDetected += (report) =>
        {
            Console.ForegroundColor = ConsoleColor.Red;
            Console.BackgroundColor = ConsoleColor.White;
            Console.WriteLine($"\n!!! 检测到游戏崩溃 !!!");
            Console.WriteLine(report);
            Console.ResetColor();
        };

        // 准备启动参数
        var param = new LauncherParam
        {
            // 基本路径设置
            GameDir = @"C:\Minecraft\.minecraft",
            LauncherName = "QomicexLauncher",

            // 游戏版本设置
            Version = "1.20.1",
            MaxMemory = "4096",

            // 窗口设置
            FullScreen = false,
            Width = "1920",
            Height = "1080",

            // Java 配置
            Java = new DataDetails.Java
            {
                Name = "Java 17",
                Path = @"C:\Program Files\Java\jdk-17\bin\java.exe",
                Version = "17.0.2",
                VersionID = 17,
                Type = "JDK",
                Arch = "x64"
            },

            // 账户配置（离线模式）
            Account = new DataDetails.Account
            {
                Name = "Steve",
                Uuid = "...", // 实际使用时需要有效的 UUID
                Token = "",
                LoginMethod = "Legacy"
            },

            // 额外 JVM 参数
            AdditionalParam = "-Djava.net.preferIPv4Stack=true"
        };

        // 启动游戏
        Console.WriteLine("正在启动 Minecraft...");
        bool success = launcher.Launch(param);

        if (success)
        {
            Console.WriteLine("游戏启动成功！正在运行中...\n");

            // 保持程序运行，等待游戏退出
            // 在实际应用中，这里应该使用 ManualResetEvent 或其他同步机制
            Console.WriteLine("按 Enter 键强制结束游戏...");
            Console.ReadLine();

            // 强制结束游戏
            launcher.KillGame();
        }
        else
        {
            Console.WriteLine("游戏启动失败！请检查日志了解详情。");
        }
    }
}

总结

Launcher 类是 Qomicex.Core 的核心组件，提供了完整的 Minecraft 游戏启动功能：

  1. 完整的启动流程 - 从参数准备到进程管理的完整流程
  2. 事件驱动架构 - 通过事件提供实时的游戏输出和状态反馈
  3. 崩溃检测 - 实时监控和检测游戏崩溃，提供崩溃报告
  4. 版本兼容 - 支持各种 Minecraft 版本和 Mod 加载器的参数生成
  5. 日志管理 - 自动记录每日日志，方便问题排查
